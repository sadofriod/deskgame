# 01 实体与聚合

## 1. Room 聚合

**聚合根**：Room

**核心属性**
- roomId
- ownerOpenId
- gameState: wait | start | end
- ruleSetCode
- deckTemplateCode
- playerCount
- currentFloor
- currentStage
- currentMatchId

**不变量**
- playerCount 必须在 5-10。
- gameState = start 时禁止新玩家加入。
- currentFloor 范围为 1-8（终局可标记为 9）。
- currentStage 只能按既定阶段顺序推进。

## 2. MatchPlayer 实体

**核心属性**
- openId
- identityCode
- chosenRoleCode（preparation 阶段确认前可为空）
- currentHp（确认角色前可为空）
- maxHp（确认角色前可为空）
- isAlive
- canSpeak
- canVote
- voteModifier
- status

**不变量**
- chosenRoleCode 为空时，只允许停留在 preparation 阶段。
- currentHp <= 0 时，默认必须 isAlive = false。
- isAlive = false 时不可提交行动牌或投票。
- 空押或被禁投时不可投票；被禁言时不可发言。

## 3. Match 聚合

**聚合根**：Match

**核心职责**
- 记录回合列表与阶段推进轨迹。
- 产出复盘可读结构。

**核心属性**
- matchId
- roomId
- rounds
- winnerResult

## 4. Round 值对象

**核心属性**
- floor
- environmentCard（揭示前可为空）
- roundKind（揭示前可为空）
- currentVoteRound
- actionSubmissions
- damages
- voteSubmissions
- settlementResult
- voteResult
- stageSnapshots
