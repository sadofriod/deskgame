# 01 实体与聚合

## 1. Room 聚合

**聚合根**：Room

**核心属性**
- roomId
- roomCode
- ownerOpenId
- gameState: wait | selecting | playing | ended
- playerCount
- roomConfig
- currentRound
- currentStage
- envDeck: 环境牌序列

**不变量**
- playerCount 必须在 5-10。
- 大厅阶段的玩家座位号必须唯一且连续，中途退出后需要补位。
- gameState = playing 时禁止新玩家加入。
- currentRound 在大厅和身份选择阶段为 0，正式对局为 1-8。
- currentStage 只能按既定阶段顺序推进。
- 仅房主可更新房间配置或解散房间。

## 2. Player 实体

**核心属性**
- openId
- nickname
- avatar
- seatNo
- candidateRoles
- selectedRole
- hp（默认 4）
- votePower（默认 1）
- isAlive
- selectedAction
- passedBet
- voteTarget
- isReady
- joinTime

**不变量**
- hp <= 0 必须 isAlive = false。
- isAlive = false 时不可选择身份、押牌或投票。
- selectedRole 一旦确认不可修改。
- 本回合选择不押牌时，不可发言且 votePower = 0。

## 3. Match 聚合

**聚合根**：Match

**核心职责**
- 记录回合列表、阶段推进轨迹与复盘摘要。
- 产出复盘可读结构。

**核心属性**
- matchId
- roomId
- rounds
- winnerResult
- replaySummary

## 4. Round 值对象

**核心属性**
- round
- environmentCard
- betSubmissions
- actionLogs
- voteSubmissions
- voteResult
- settlementResult
- stageSnapshots
