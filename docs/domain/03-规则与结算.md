# 03 规则与结算

## 1. 票权计算

**核心公式**
- finalVotePower = baseVotePower(1) + actionBonus

**行动牌加成**
- 骂：+0.5
- 其他行动牌：+0
- 未押牌 / 已出局：finalVotePower = 0

**平票处理**
- 首轮投票平票时进入额外辩论环节。
- 额外辩论结束后进入 voteRound = 2 的受限重投。
- 平票玩家不能投票。
- 仅剩余玩家在平票玩家之间二选一投票。
- 第二轮仍平票，则本层无人因投票出局。

## 2. 伤害结算

**基础公式**
- finalHp = currentHp - totalDamage
- totalDamage = envDamage + actionDamage
- hp <= 0 标记出局

**伤害来源**
- 环境伤害：由 EnvironmentCard 决定。
- 行动伤害：吹 / 抓 / 吸（有屁回合额外 +1）。

**免疫与减伤**
- 忍：本回合免疫可防御伤害。
- 抓：不可防御伤害，默认忽略普通防御；特殊角色技能可覆写。

## 3. 回合类型判定

- 有屁回合：EnvironmentCard 为有屁 / 有臭屁 / 有闷屁。
- 无屁回合：EnvironmentCard 为无屁。

## 4. 胜利判定

**实时判定条件**
- 乘客胜利：屁者人数 = 0；或第 8 层结束且存活乘客 > 存活屁者。
- 屁者胜利：任意时刻存活乘客 <= 存活屁者；或活跃人数 = 0；或累计完成 4 轮有屁回合。

## 5. 流程约束

- Round 在 preparation 阶段创建为空壳，environment 阶段补齐 environmentCard 与 roundKind。
- 行动牌提交后不可修改。
- 同一 floor 允许多条 ActionSubmission，用于记录押牌与行动阶段直出。
- actionCard 为空不可投票与发言。
- 同一 floor 最多允许 2 轮 VoteSubmission，由 voteRound 区分。
- preparation 只有在全部玩家确认角色后才允许推进到 bet。
- 阶段不可回退，系统仅允许顺序推进；tieBreak -> vote 是唯一回环。
