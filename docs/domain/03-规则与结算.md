# 03 规则与结算

## 1. 房间与准备规则

- 房主是首个进入房间的玩家，拥有配置管理与解散房间权限。
- 房间人数范围为 5-10，玩家按进入顺序分配座位号。
- 大厅阶段允许调整人数和扩展职业配置。
- 所有玩家准备完成后，系统自动进入身份选择阶段。

## 2. 身份选择规则

- 每位玩家获得 3 个候选角色，只能确认 1 个。
- 角色确认后不可更改，未完成选择的玩家不能进入下一阶段。
- 屁者角色可额外查看同阵营玩家序号与环境牌排序规则。
- 非屁者角色仅能查看自己的角色信息和技能说明。

## 3. 票权计算

**核心公式**
- finalVotePower = baseVotePower(1) + actionBonus

**行动牌加成**
- 骂：+0.5
- 其他行动牌：+0
- 不押牌 / 已出局：finalVotePower = 0

**平票处理**
- 首轮平票时触发重新投票。
- 重新投票后仍平票，则本回合无人出局。

## 4. 伤害结算

**基础公式**
- finalHp = max(0, currentHp - totalDamage + totalHeal)
- totalDamage = envDamage + actionDamage + penaltyDamage
- hp <= 0 标记出局

**伤害来源**
- 环境伤害：由 EnvironmentCard 决定。
- 行动伤害：吹 / 抓 / 吸（有屁回合额外 +1）。
- 不押牌惩罚：本回合无发言权、无投票权，并在结算时承担额外伤害。

**免疫与减伤**
- 忍：本回合免疫全部伤害。
- 抓：不可防御伤害，忽略免疫。

## 5. 回合类型判定

- 有屁回合：EnvironmentCard 为有屁 / 有臭屁 / 有闷屁。
- 无屁回合：EnvironmentCard 为无屁。

**回合效果摘要**
- 有屁回合：所有存活玩家先承受 1 点基础伤害，再叠加行动效果。
- 无屁回合：无基础伤害，仅结算行动牌带来的收益或惩罚。

## 6. 胜利判定

**实时判定条件**
- 乘客胜利：屁者人数 = 0；或第 8 层结束且存活乘客 > 存活屁者。
- 屁者胜利：任意时刻存活乘客 <= 存活屁者；或乘客阵营被清空。
- 平局：活跃人数 = 0。

## 7. 流程约束

- 完整流程为：建房 -> 准备 -> 身份选择 -> 8 回合对局 -> 胜负判定 -> 复盘。
- 押牌提交后不可修改。
- 不押牌玩家不可投票与发言。
- 所有玩家完成押牌后才揭示环境牌并进入玩家行动阶段。
- 阶段不可回退，系统仅允许顺序推进。
- 任一阵营提前满足胜利条件时立即终止对局。
