核心业务流程（优化版）
本流程为小程序核心业务逻辑，按建房→身份选择→8 回合对局→胜负判定全链路设计，优化表述更贴合技术实现与用户交互逻辑，具体流程如下：
2.1 建房阶段（游戏初始化）
2.1.1 流程定义
房主作为首个进入房间的用户，拥有房间配置管理权与房间解散权；其余玩家按进入顺序分配唯一序号（1、2、3...），中途退出的玩家由后续进入的玩家顺位补全，确保房间人数稳定。
2.1.2 交互逻辑
1. 房主点击「创建房间」，系统自动生成 6 位唯一房间码，同步创建房间实例，初始化房主权限；
2. 玩家可通过扫码加入（扫描房间二维码）或手动输入房间码两种方式进入房间；
3. 房间内实时显示玩家列表（含昵称、头像、序号），房主可在配置区修改游戏人数（5-10 人）、扩展职业开关等基础配置；
4. 所有玩家进入房间后，需点击「准备」按钮，系统监听全员准备状态，满足条件后自动触发身份选择阶段；
5. 房主可随时点击「解散房间」终止对局，仅房主可见该权限。
2.1.3 技术实现要点
- 房间码生成：采用随机数 + 唯一性校验算法，确保 6 位房间码不重复；
- 玩家序号分配：按玩家进入房间的时间戳排序，自动分配 1-N 的唯一序号；
- 权限管控：通过微信 OpenID 区分房主与普通玩家，前端渲染时动态显示权限按钮；
- 状态监听：前端实时轮询 / 云数据库实时推送，监听全员准备状态。
2.2 身份选择阶段（角色分配）
2.2.1 流程定义
玩家进入身份选择界面后，系统根据房间人数与配置，随机生成 3 个候选角色牌，玩家从中选择 1 个作为自己的角色。选择完成后，角色头像自动填充至游戏界面的人物头像位置，点击头像可查看角色技能介绍（不显示身份信息）。
2.2.2 核心权限规则
- 屁者角色：选择后可查看同阵营屁者的玩家序号，并获取环境牌排序规则，用于制定对局策略；
- 非屁者角色：仅可查看自身角色信息，无额外权限，保障信息公平性；
- 选择限制：玩家仅能选择 1 个角色，选择后不可更改，未选择状态下无法进入下一阶段。
2.2.3 技术实现要点
- 角色牌生成：按房间人数 / 配置动态生成 3 个候选角色，确保角色配比符合桌游规则（如 5 人房：1 屁者 + 4 乘客）；
- 信息加密：屁者阵营信息仅对屁者可见，通过云数据库权限管控实现，普通玩家无法查询·；
- 头像与技能渲染：前端根据角色 ID 动态加载头像与技能文案，隐藏身份标识；
- 选择状态同步：玩家选择后实时更新云数据库，系统监听全员选择状态，触发下一阶段。

2.3.2 回合核心流程（单回合 6 阶段）
单回合包含押牌阶段→玩家行动阶段→伤害结算阶段→发言投票阶段，完成后自动进入下一回合，直至 8 回合结束 / 阵营提前获胜 / 全员淘汰。
阶段 1：押牌阶段（核心交互阶段）
1. 系统自动触发押牌阶段，底部操作区显示 6 个行动牌按钮 +「不押牌」按钮；
2. 玩家需选择 1 个行动牌提交，或选择「不押牌」，选择后不可更改；
3. 系统实时监听全员押牌状态，所有玩家完成选择后，自动进入环境牌展示阶段；
4. 特殊规则：选择「不押牌」的玩家，本回合无投票权、无发言权，需在伤害结算阶段承受额外伤害。
阶段 2：玩家行动阶段（效果执行阶段）
1. 系统随机选取 1 名存活玩家作为行动起始者，按玩家序号依次执行；
2. 每位玩家执行时，系统展示其押牌信息，根据行动牌规则触发对应效果（如「吹」指定身边玩家加伤、「抓」指定玩家加伤）；
3. 本阶段不设置计时，待所有玩家完成行动后，自动进入伤害结算阶段；
4. 效果执行期间，前端实时同步玩家状态变化，保障交互流畅。
阶段 3：伤害结算阶段（数据计算阶段）
1. 系统获取当前回合类型（有屁 / 无屁），结合所有玩家的押牌信息，自动计算伤害、免疫、减伤效果；
2. 核心计算逻辑：
  - 有屁回合：所有存活玩家先承受 1 点基础伤害，再根据行动牌调整最终伤害；
  - 无屁回合：无基础伤害，仅按行动牌效果触发特殊效果；
3. 玩家最终血量更新为：最终血量 = 初始血量 - 伤害 + 恢复血量，血量最低为 0；
4. 血量为 0 的玩家，系统自动标记为出局状态（头像置灰、禁用操作）。
阶段 4：发言投票阶段（淘汰判定阶段）
1. 系统随机选取 1 名存活玩家作为发言起始者，按玩家序号依次发言，本阶段不设置计时；
2. 所有玩家完成发言后，进入投票环节，存活玩家需选择 1 名目标玩家投票；
3. 票权计算规则：
  - 基础票权：1 票 / 人；
  - 额外加成：选择「骂」行动牌的玩家，票权 + 0.5 票；
  - 无效票权：未押牌 / 已出局玩家，票权为 0；
4. 投票结果统计：
  - 系统按票权加权计算各玩家得票，找出得票最高者；
  - 若票数相同，触发重新投票；重新投票后仍平票，则无人出局；
  - 得票最高者，系统自动标记为出局状态。
2.4 流程推进规则
1. 单回合完成顺序：押牌→玩家行动→伤害结算→发言投票，完成后自动进入下一回合的押牌阶段；
2. 终止条件：
  - 8 个回合全部完成，系统触发最终胜负判定；
  - 任意阵营提前满足胜负条件，立即终止对局，公布结果；
  - 所有玩家被淘汰，对局提前结束。

2.5.2 游戏复盘
对局结束后，系统自动生成复盘数据，支持所有玩家查看，复盘内容包含：
1. 玩家最终信息：每位玩家的角色、最终血量、存活状态；
2. 回合详情：每回合的押牌信息、行动效果、伤害结算、投票结果；
3. 胜负关键节点：重点出局玩家、阵营人数变化关键回合；
4. 对局统计：总回合数、存活玩家数、票权分布等数据。
