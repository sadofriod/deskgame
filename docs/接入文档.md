# DeskGame Backend 接入文档

## 1. 核心流程（对齐最新流程文档）

当前阶段流转：

`preparation -> bet -> environment -> action -> damage -> talk -> vote -> [tieBreak] -> settlement`

平票时会进入分支：

`vote -> tieBreak -> vote`

其中：

- `preparation`：发牌、候选角色确认（`RoleSelectionStarted` / `RoleSelectionCompleted`）
- `bet`：玩家提交本层押牌（通过 `SubmitAction` / `POST /rooms/:roomId/actions`，`sourceStage=bet`）
- `environment`：揭示当层环境牌（`RevealEnvironment`）
- `action`：执行行动阶段动作（`SubmitAction`，`sourceStage=action`）
- `damage`：伤害结算
- `talk`：发言阶段
- `vote`：投票阶段
- `tieBreak`：平票重投阶段（限制投票目标）
- `settlement`：结算淘汰、胜负判定与层数推进

参考：[`docs/flows/room-flow.zh-CN.md`](./flows/room-flow.zh-CN.md)

## 2. HTTP API

基础地址示例：`http://localhost:3000`

### 2.1 房间相关

1. `POST /rooms` 创建房间  
   body 关键字段：
   - `ownerOpenId: string`
   - `ruleSetCode: string`
   - `deckTemplateCode: string`
   - `requestId?: string`

2. `POST /rooms/:roomId/players` 加入房间  
   body 关键字段：`openId`、`nickname`、`avatar?`、`requestId?`

3. `DELETE /rooms/:roomId/players/:openId` 离开房间  
   body 可选：`requestId?`

4. `POST /rooms/:roomId/start` 开始游戏（房主）

5. `POST /rooms/:roomId/role-selection` 确认角色  
   body 关键字段：`openId`、`roleCode`

6. `POST /rooms/:roomId/actions` 提交行动/押牌  
   body 关键字段：`openId`、`cardInstanceId`  
   > 在 `bet` 阶段用于押牌，在 `action` 阶段用于行动。

7. `POST /rooms/:roomId/environment/reveal` 揭示环境牌  
   body 建议传：`openId`（房主）

8. `POST /rooms/:roomId/votes` 提交投票  
   body 关键字段：`openId`、`voteRound?`、`voteTarget?`、`votePowerAtSubmit?`

9. `POST /rooms/:roomId/stage/advance` 推进阶段（房主指令）

10. `GET /rooms/:roomId` 获取房间快照

### 2.2 用户相关

- `POST /users` 创建/更新用户
- `GET /users/:openId` 获取用户
- `PUT /users/:openId` 更新用户

## 3. WebSocket 接入

客户端发送事件名固定为 `command`，消息结构：

```json
{
  "type": "COMMAND",
  "name": "CreateRoom",
  "requestId": "req-001",
  "payload": {}
}
```

服务端返回：

- `event`：领域事件广播
- `error`：命令错误（如参数缺失、阶段非法）

### 3.1 支持的命令名

- `CreateRoom`
- `JoinRoom`
- `LeaveRoom`
- `StartGame`
- `ConfirmRoleSelection`
- `SubmitAction`
- `RevealEnvironment`
- `SubmitVote`
- `AdvanceStage`

## 4. 联调建议

- 房间主键请使用 `room.roomId`（不是 `room.id`）。
- 前后端联调至少监听：`RoleSelectionStarted`、`RoleSelectionCompleted`、`EnvironmentRevealed`、`RoundSettled`、`VoteResolved`、`WinnerDecided`。
